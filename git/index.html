Distributed

With Git you clone the whole history of the project from inception to current state

git clone username@host:/path/to/repository

After the cloning there is basically no information that you do not have compared to the server you cloned from

With SVN you checkout only a specific snapshot of the project

svn checkout https://path/to/repository

With Git you have the whole history locally

Repository from scratch

With Git you add files to the so called 'index' which is basically a staging area.

Git internals

Git is "content-addressable filesystem with a VCS user interface written on top of it"

- The key being the checksum of the content that you committed.

This checksum is actually a SHA1 hash digest of the content of the committed files with metadata(message, author, etc) and reference to the commit key before it.

- The content being an object containing the tree structure with a snapshot of the files that you committed.

key: 78c61dcff33b0cea7755eb71c8b188af37c97b48

content: index.html, css/style.css, js/site.js

Git stores the full content of the committed files(with references to unchanged files between versions).

It does not store just the deltas like SVN. This means some speed coming from the fact that when you pull out a version it's fast since it does not have to build up the files processing all the deltas.
