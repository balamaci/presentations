Little History

Designed it as a replacement for BitKeeper

Linus hates CSV and SVN with passion

Did research looking for a solution that :

  • Supported distributed workflows
  • Safeguard against data corruption either accidental or malicious
  • Had high performance(was very fast)

Development started and on 3 Apr 2005 and in 26 June 2005 they were using Git to manage the release for the Linux kernel

Distributed

With Git you clone the whole history of the project from inception to current state

git clone username@host:/path/to/repository

After the cloning there is basically no information that you do not have compared to the server you cloned from

With SVN you checkout only a specific snapshot of the project

svn checkout https://path/to/repository

With Git you have the whole history locally

Non Distributed

Single central repository from where you checkout and commit

Distributed

Central repository can still exist but whole new workflows can appear
